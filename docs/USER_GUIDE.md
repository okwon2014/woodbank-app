## 1. 한눈에 보기

- **무엇을 하는 앱인가**: 야외에서 나무 시료를 채취하면서 위치·수종·계측을 기록하고 사진 4종을 찍어 서버에 동기화합니다. 그 시료가 다듬어져 만들어지는 디스크·블록·슬라이드·DNA 추출물 등도 같은 시스템에 등록해 끝까지 추적합니다.
- **어디서 쓰나**: 모바일 브라우저(PWA), 데스크톱 브라우저 모두. 인터넷이 끊겨도 야장 입력은 가능하고 자동 동기화됩니다.
- **로그인은 한국어 이메일 계정으로**. 가입 직후엔 권한이 없어 운영 책임자가 역할을 부여해야 데이터가 보입니다.

## 2. 역할과 권한

5가지 역할 — 운영 책임자(admin)가 부여합니다.

| 역할 | 무엇을 할 수 있나 |
|---|---|
| **admin** | 모든 데이터 읽기·쓰기, 사용자/지역 관리, 일괄 등록·내보내기, 시편/DNA 등록, 통계 전체 |
| **lead** (조사책임자) | 담당 시군구의 모든 데이터 읽기·쓰기. 시편/DNA 결과 등록 가능. 일괄 등록·내보내기 가능 |
| **surveyor** (조사원) | 담당 시군구 데이터 읽기. 새 야장 등록(본인이 surveyor_id). 본인 등록 24시간 내 편집 가능 |
| **collaborator** (외부 협력자) | 공유받은 특정 조사지점만 읽기 |
| **guest** (역할 미배정) | 가입은 됐지만 아무 데이터 안 보임. admin 의 역할 부여 대기 상태 |

> 권한 부족 시 친절한 안내 화면(`/forbidden`)이 떠서 어떤 역할이 필요한지·연락처(admin 이메일)를 보여줍니다.

## 3. 시작하기 — 로그인·프로필

### 가입과 로그인

1. 운영 도메인(예: <https://woodbank-app.vercel.app>) 의 `/login` 접속.
2. 비밀번호 로그인 또는 **매직 링크**(이메일로 받은 링크 클릭) 둘 다 가능.
   - 비밀번호를 잊었으면 「비밀번호 잊으셨나요?」 → 이메일로 재설정 링크가 옵니다.
3. **신규 가입자**: 가입 직후 역할이 `guest`. 운영 책임자에게 역할 부여를 요청하세요(이메일은 `/forbidden` 페이지에 안내).

### 프로필 확인

`/profile` — 표시 이름·소속·현재 역할·담당 지역(시군구 코드) 확인. 담당 지역이 비어 있으면 lead/surveyor 권한이 있어도 데이터가 안 보이니 운영 책임자에게 매핑 요청.

## 4. 현장 작업 — 야장 입력

`/events/new` (메뉴 「+ 새 야장」)에서 한 시료를 처음부터 끝까지 입력합니다.

### 입력 순서

1. **채취 기본**: 채취번호(`2025_담양_04`), 채취일. 「자동」 버튼으로 같은 site_code 의 다음 번호 제안.
2. **조사 지점**: 지점 코드(`2025_담양`), 시도·시군구·시군구 코드(`46710`), 장소 상세, 지형.
3. **위치**:
   - 「📍 현재 위치 가져오기」로 GPS 측정.
   - 「🏷 좌표로 주소 채우기」 클릭하면 위·경도에서 행정구역과 장소 상세를 자동으로 채워 줍니다(빈 칸만 채움).
   - GPS 정확도가 ±30 m 초과면 노란 경고, ±100 m 초과면 빨간 경고. 야외 개활지로 이동해 다시 측정 권장.
4. **수종·계측**: 수종 선택, 수고(m), DBH(cm). 통상 범위를 벗어나면(예: 수고 200 m) 즉시 안내 메시지.
5. **DNA 시료**: 「DNA 시료를 채취하였음」 체크 + DNA 라벨(선택). **이건 채취 단계 정보**일 뿐, 분석 결과는 나중에 시편 단계에서 등록합니다.
6. **사진 4종**: 수형·수피·가지·잎/낙엽. 카메라 직접 촬영 또는 갤러리 선택. 단말에서 자동 압축(기본 1600 px·85 % JPEG). 화질은 「빠름/보통/고화질」 토글로(단말별 기억).
7. **특기사항**: 자유 기록.
8. **저장 + 동기화**. 온라인이면 즉시 서버 동기화, 오프라인이면 단말 큐에 보존.

### 안내가 뜨는 경우들

- ⚠️ **수고/DBH/방위/고도 이상치**: 입력값이 통상 범위 밖. 단위 또는 입력값 확인.
- ⚠️ **시군구 코드가 마스터에 없음**: 오타 의심.
- 🌳 **반경 5 m 안에 이미 등록된 다른 개체목**: 같은 나무를 다시 등록하는지 확인하라는 안내.
- ✓/⚠️ **채취번호 중복**: 입력 중 디바운스로 서버 확인. 빨간 경고 떴으면 다른 번호 사용.

## 5. 오프라인 동작과 동기화

- **새 야장 등록**은 비행기 모드에서도 동작. 사진까지 모두 단말 IndexedDB에 보존.
- 온라인 복귀 시 자동 동기화 — 페이지 포커스 복귀 / 5 분 타이머 / Android Chrome 의 Background Sync 가 깨워서 시도.
- `/queue` (메뉴 「동기화 큐」) 에서 대기 중인 항목·재시도·에러 확인.

### 큐가 안 줄어들 때

- **재시도 5회 초과 → 자동 재시도 중단** 상태. 원인(보통 RLS 권한·필수 컬럼 누락) 해결 후 [지금 재시도] 또는 [큐에서 제거].
- **서버 충돌 빨간 배지** — `sample_no` 중복 등 unique 제약 위반. 채취번호를 고친 새 야장으로 저장하거나 [큐에서 제거].
- **편집/사진 추가/삭제는 오프라인 미지원** — 새 야장 등록만 큐 사용. 편집 흐름은 즉시 서버 호출이라 오프라인이면 실패.

## 6. 야장 보기·편집·삭제

- **`/events`** — 본인 권한 범위 내 야장 목록. 수종·시군구·날짜·검색어 필터.
- **`/events/[id]`** — 야장 상세. 채취 기본·개체목·지점·사진(클릭 시 라이트박스로 원본+EXIF)·시편 트리.
- **편집** — admin/lead 는 언제든, surveyor 본인은 등록 후 24시간 내. 사진 추가/삭제는 「수정 → 사진」.
- **삭제** — admin 만. 자식 시편/사진도 함께 삭제됨(되돌릴 수 없음).

## 7. 지도

`/sites?view=map` 에서 권한 내 모든 개체목을 OSM 지도에 표시합니다.

- 빨간 마커(라벨: `site_code #tree_local_no`) 클릭 → 그 개체목 상세로 이동.
- **🖼 이미지 저장** (좌상단) — 현재 화면을 PNG 로 받음. 마커·라벨·OSM 라이선스 모두 합성되어 보고서에 그대로 사용 가능.
- 마우스 위치의 좌표·줌이 우하단에 실시간 표시 — 데이터 검증·외부 좌표 대조에 유용.
- 페이지 하단에 좌표 범위(bounding box) 와 「한반도 밖 좌표 N건」 경고(잘못 입력된 데이터 즉시 식별).

## 8. 시편 추적 — 디스크·블록·슬라이드 등

야장은 현장 기록의 끝이고, 거기서 만들어지는 모든 시편을 같은 시스템에 등록해 추적합니다.

### 시편 종류 9가지

| 코드 | 이름 | 설명 |
|---|---|---|
| D | 디스크 | 줄기/가지를 가로로 잘라낸 원판 |
| C | 증분코어 | Increment borer 등으로 채취한 시추 |
| B | 블록 | 디스크/코어에서 잘라낸 작은 토막 |
| L | 현미경 슬라이드 | 박편·영구 슬라이드 |
| T | 연륜표본 | 연륜 측정·교차연대측정용 |
| F | 해리 섬유 | 해리된 단섬유 표본 |
| X | 추출물 | DNA·화학 추출물 |
| R | 잔여 보존 | 분석 후 남은 보관용 |
| O | 기타 | 위에 없는 분류 |

### 사람용 코드 규칙

부모 코드에 점(`.`)을 찍고 종류 약자 + 2자리 순번을 붙입니다.

```
야장          2025_담양_04
└ 디스크      2025_담양_04.D01
  ├ 블록      2025_담양_04.D01.B03
  │   ├ 슬라이드    2025_담양_04.D01.B03.L02
  │   └ 해리 섬유   2025_담양_04.D01.B03.F01
  └ DNA 추출    2025_담양_04.D01.X01
```

깊이 제한 없습니다. 부모를 보고 어디서 왔는지 한눈에 파악되며, 자식이 늘어도 영향 없습니다.

### 만드는 흐름

1. 야장 상세 페이지의 **시편 (Specimens)** 섹션에서 「+ 1차 시편 추가」 (admin/lead 만)
2. 종류·보관 위치·설명 입력 → 추가하면 사람용 코드가 자동 생성.
3. 각 노드 옆 「+ 자식」 으로 더 깊은 시편 추가.
4. 사람용 코드 클릭 시 `/specimens/<id>` 상세 페이지 — 추적 경로·자식 트리·QR.

### 상태 관리

각 시편은 `active / consumed(소진) / lost(분실) / destroyed(폐기)` 4 상태. 수정은 시편 상세에서.

## 9. DNA 분석 결과

야장의 「DNA 시료 채취」 체크는 **현장 정보**일 뿐, 분석 결과는 거기 들어가지 않습니다. 분석 결과는 다음 흐름:

1. 야장 상세에서 「+ 1차 시편 → X (Extract, 추출물)」 추가
2. 그 X 시편의 `/specimens/<id>` 페이지로 이동
3. 「DNA 분석 결과」 섹션의 「+ 결과 추가」 (admin/lead)
4. 분석 마커(ITS·rbcL·matK 등) · 식별 결과 · 유사도 % · 분석 책임자 · 분석일 · 메모 · 첨부 파일(PDF/Excel/CSV 등)

여러 마커의 결과를 같은 시편에 매달 수 있습니다(N건). 단순 디스크(D)나 블록(B)에 직접 매다는 것도 가능하지만 추적상 X 시편을 권장.

## 10. 라벨 인쇄 — QR + 사람용 코드

`/specimens/print` — 시편 라벨을 한꺼번에 인쇄합니다.

진입 경로:
- 야장 상세의 「🏷 라벨 인쇄」 → 그 야장 아래 모든 시편
- 시편 상세의 「🏷 라벨 인쇄」 → 그 시편 한 장

### 두 가지 모드

| 모드 | 용도 | 설정 |
|---|---|---|
| **A4 격자** | 일반 라벨지(Avery/Formtec) | 라벨 가로·세로 (mm), 열 수, 행 수 |
| **단일 라벨** | 라벨 프린터(Brother QL · DYMO · Zebra) | `@page size = 가로 × 세로 mm`, 한 장에 한 라벨 |

라벨 내용: QR 코드 + 사람용 코드(굵은 monospace) + 종류 한글명·상태.

**QR 텍스트 옵션** — 기본은 사람용 코드(스캐너가 텍스트 반환). 「URL 로 인쇄」 체크하면 `https://<도메인>/specimens/<id>` 로 — 스마트폰 카메라로 스캔 시 그 시편 상세 페이지로 바로 이동.

## 11. 통계 대시보드

`/stats` (메뉴 「통계」) — 본인 권한 범위 내 데이터의 통계.

표시 항목:
- KPI 카드 5개: Sites · Trees · Events · Photos · DNA 분석 결과
- 수고·DBH 요약 (N / 평균 / 중위수 / 표준편차 / 최소 / 최대)
- 수고·DBH 분포 히스토그램
- 시군구별/시도별 야장 수 (상위 20)
- 수종별 야장 수 (상위 20) + 평균 측정값 표 (상위 15)
- 월별 야장 등록 시계열 (최근 12개월)
- DNA 채취 비율 (야장의 채취 여부 기준, 분석 결과와 별개)
- 동기화 상태 분포

## 12. 내보내기 — Excel·CSV·Word·PDF·ZIP

`/admin/export` (admin/lead 만)에서 필터 적용 후 다운로드.

| 형식 | 적합한 용도 |
|---|---|
| **Excel (.xlsx)** | 분석·필터링. 24컬럼 + 사진 수 |
| **CSV (UTF-8 BOM)** | R · Python · DB import. Excel 도 호환 |
| **Word (사진 포함)** | 1샘플 = 1페이지 보고서. 사진 4종 110×110px 임베드 |
| **Word (사진 없이)** | 빠른 텍스트 보고서 |
| **PDF (인쇄 뷰)** | 새 탭에서 브라우저 인쇄 → PDF 저장. A4 18 mm 마진 |
| **📦 ZIP** | 사진 원본 + queue.json + README. `/admin/import` 의 「ZIP 가져오기」로 복원 가능 |

## 13. 가져오기 — 일괄 등록 (TSV/ZIP)

`/admin/import` (admin/lead) — 외부 데이터를 일괄 등록.

### TSV (Excel/Google Sheets 붙여넣기)

1. 헤더가 포함된 표를 선택해 복사 → 텍스트 영역에 붙여넣기.
2. **헤더는 영문·한국어 둘 다 인식** — `/admin/export` 의 Excel/CSV 를 손대지 않고 그대로 가져올 수 있음.
3. 미리보기에서 ❌(오류) / ⚠️(경고) / ✅(정상) 확인. 같은 `sample_no` 가 이미 있으면 덮어쓰기.

### ZIP

- `/admin/export` 나 `/queue` 에서 받은 백업 ZIP 을 그대로 가져옴.
- `queue.json` 만 파싱하고 `pdf/docx/이미지/기타 모든 파일은 무시` — 사진은 야장 등록 후 개별 첨부.
- device 백업(`/queue` 큐 백업) · server 백업(`/admin/export` ZIP) 두 schema 모두 지원.

## 14. 큐 백업 (단말 데이터 보존)

`/queue` 의 「📦 백업 ZIP」 — 단말 IndexedDB 의 동기화 대기 데이터를 한 파일로.

ZIP 내용:
- `queue.json` — 들여쓰기 2칸 JSON (사람이 텍스트로 읽기 좋음). 야장·사진 메타·큐 상태·사이트/개체목 캐시
- `photos/<event_id>/<photo_id>.jpg` — 원본 JPEG
- `README.txt` — 파일 구조 안내

용도: 단말 분실/교체 전 백업, 동기화 못한 데이터를 외부에서 분석, 분쟁/감사 시 원본 보존.

복원: 받은 ZIP 을 `/admin/import` 의 「ZIP 가져오기」 로(메타데이터). 사진은 야장 상세에서 개별 첨부.

## 15. 관리자(admin) 작업

`/admin` (admin 만) 대시보드의 빠른 링크:

- **📊 통계 대시보드** — 본인 권한 범위(admin은 전체).
- **📥 일괄 등록** — `/admin/import` (admin/lead).
- **📤 일괄 다운로드** — `/admin/export` (admin/lead).
- **사용자 관리** — `/admin/users`.

### `/admin/users` — 사용자/역할/지역

- 「+ 새 사용자 초대」 — 이메일로 가입 링크 발송 (`SUPABASE_SERVICE_ROLE_KEY` + `NEXT_PUBLIC_SITE_URL` 환경변수 필요).
- 각 사용자 행 옆에서 역할 변경 · 활성/비활성 토글 · 담당 시군구 매핑 추가/제거.
- 「역할 미배정 사용자 N명」 노란 배너 — guest 사용자 대기 알림.

### 신규 가입 이메일 알림 (선택)

운영자에게 새 가입자가 생겼을 때 자동 이메일을 받을 수 있습니다. 셋업 방법은 [DEPLOYMENT.md](DEPLOYMENT.md) §3.

## 16. 모바일 PWA — 홈 화면에 추가

운영 도메인에 모바일 브라우저로 접속한 뒤:

- **iOS Safari**: 공유 버튼 → 「홈 화면에 추가」.
- **Android Chrome**: 우측 메뉴 → 「앱 설치」 또는 「홈 화면에 추가」.

설치 후엔 단독 앱처럼 동작 — 주소창 없이, 캐시된 화면이 즉시 뜨고, 오프라인에서도 새 야장 입력 가능.

**Background Sync**: Android Chrome/Firefox는 OS가 인터넷 복귀를 감지해 자동으로 큐 동기화를 시도합니다. iOS Safari 는 미지원 — 대신 페이지 포커스 복귀 / 5분 타이머 / online 이벤트 가 안전망.

## 17. 트러블슈팅 (자주 마주치는 상황)

| 상황 | 원인·해결 |
|---|---|
| **로그인은 되는데 모든 목록이 비어 있다** | `users_meta.role = 'guest'`. admin 이 역할 부여 + 담당 지역 매핑 필요 |
| **`/admin` 또는 보호 페이지 진입 시 `/forbidden`** | 권한 부족. 안내 페이지에 필요한 역할·현재 역할·admin 연락처 표시 |
| **사진 업로드 실패: "Bucket not found"** | 운영자에게 알림 — `photos` 버킷 미생성(003 마이그레이션 누락) |
| **시편 「+ 자식」 클릭 시 동시에 같은 코드 생성 에러(23505)** | 다른 사용자가 같은 부모에 동시 추가. 한 번 더 「+ 자식」 누르면 다음 번호로 진행 |
| **동기화 큐가 안 줄어듦** | `/queue` 에서 `last_error` 확인. 보통 RLS 권한·필수 컬럼 누락·네트워크 |
| **매직링크 클릭했는데 로그인 안 됨** | 1) 링크는 한 번만 사용 가능, 만료 시간 1시간. 2) 운영자에게 Supabase Redirect URLs 등록 확인 요청 |
| **PWA 설치 안 됨** | HTTPS 필요. 로컬 `localhost` 면 가능, 외부는 운영 도메인에서. iOS는 Safari 만 |
| **편집/사진 추가가 오프라인에서 실패** | 의도된 한계. 새 야장 등록만 오프라인 큐 사용 |
| **시편 상세에 DNA 분석 결과가 안 보임** | 운영자에게 알림 — 마이그레이션 006/008 적용 확인 필요 |
