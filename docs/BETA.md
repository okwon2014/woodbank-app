# Woodbank 베타 안내

이 문서는 **내부 베타 시험**(기능·안정성 점검 단계)에 참여하시는 분께 드리는 안내입니다. 일반 사용 매뉴얼이 아니라 "지금 어디까지 되고, 어디는 아직 부족한지" 정직하게 알려드리는 문서예요.

> 운영 도메인: <https://woodbank-app.vercel.app>

## 1. 베타 단계가 검증하려는 것

- **현장 입력 흐름**: 사이트·개체목·야장을 모바일에서 1건 입력해서 동기화될 때까지의 경로가 매끄러운지
- **오프라인 동작**: 비행기 모드에서 야장을 저장하고, 온라인 복귀 시 자동 동기화가 잘 되는지
- **역할별 화면**: surveyor·lead·admin 각자가 보는 화면이 자신의 권한 안에서 합리적인지
- **데이터 일관성**: 입력한 값이 RLS 안에서 일관되게 보이는지, 다운로드한 Excel/CSV/Word 가 그 값과 일치하는지

## 2. 골든 패스 (한 바퀴 직접 돌아보기)

다음 순서로 한 번 끝까지 가보시면 베타의 핵심을 모두 만지게 됩니다.

| # | 화면 | 확인할 동작 |
|---|---|---|
| 1 | `/login` | 본인 이메일·비밀번호로 로그인. 비밀번호 잊었으면 「비밀번호 재설정」 |
| 2 | `/profile` | 표시 이름·소속 확인. 담당 지역(시군구 코드)이 비어 있으면 운영자에게 매핑 요청 |
| 3 | `/events/new` | 새 야장 입력: 「📍 현재 위치」 → 「🏷 좌표로 주소 채우기」 → 채취번호 「자동」(✓ 표시 확인) → 수종·계측·DNA 입력 → 사진 4종 → 저장 |
| 4 | `/queue` | 동기화 대기열이 보였다가 사라지는지 |
| 5 | `/events` → 방금 만든 야장 클릭 | 입력값·사진이 그대로 보이는지. admin/lead 면 DNA 결과 추가도 |
| 6 | `/sites?view=map` | 지도에서 마커가 보이고, 클릭 시 트리 상세로 이동. 좌상단 **🖼 이미지 저장** 으로 현재 화면을 PNG로 받을 수 있습니다(보고서용, OSM 라이선스 포함). |
| 7 | `/admin/export` (admin·lead 만) | 필터 적용 후 Excel·CSV·Word·PDF 다운로드. 「📦 ZIP」으로 사진 + JSON 한 파일 백업도 가능 — 같은 ZIP을 `/admin/import` 의 「📦 ZIP 가져오기」로 복원 |
| 7.5 | `/stats` | 본인 권한 안의 통계 대시보드 — 지역/수종/수고·DBH 분포, 월별 추이, DNA 채취율 |
| 8 | (모바일) 비행기 모드 → `/events/new` 입력 → 저장 → 비행기 모드 해제 → 자동 동기화 확인 |

각 단계에서 어색하거나 막힌 부분이 있으면 짧게 메모해 두세요. 다음 단계의 입력값이 되거든요.

## 3. 알려진 한계 (지금은 의도된 한계)

베타 단계에서 다음은 **미구현** 또는 **부분 구현**입니다. 동작 안 한다고 버그로 보고하지 않으셔도 됩니다.

- **PWA 아이콘** — 임시 단색 로고입니다(브랜드 컬러 + "W"). 정식 디자인이 들어오면 교체합니다.
- **에러 메시지 한국어화** — 일부 시스템 에러는 영문 그대로 노출됩니다(예: `Could not find ...`). 한국어 친화 메시지는 점진적으로 적용 중.
- **검색·필터 페이징** — 야장 목록·내보내기는 한 번에 500건까지. 더 많아지면 별도 작업.
- **Sentry 등 에러 트래킹** — 미설치. 베타 중 문제 발생 시 화면 캡처 + 시각 + 어떤 동작 후 일어났는지 알려주시면 됩니다.
- **Background Sync** — Android Chrome / Firefox 에서만. iOS Safari 는 기존 안전망(online 이벤트 + 5분 타이머 + 페이지 포커스 복귀)으로 동작.
- **편집·사진 수정의 오프라인** — 새 야장 등록만 오프라인 큐를 사용합니다. 기존 야장 *편집* 과 *사진 추가/삭제* 는 즉시 서버 호출이라 오프라인에선 실패합니다.
- **큐 백업 ZIP** — `/queue` 페이지의 「📦 백업 ZIP」 으로 단말에 대기 중인 야장·사진을 한 파일로 받을 수 있습니다(queue.json + photos/*.jpg + README.txt). 서버 장애·단말 교체 대비. 자동 복구 기능은 아직 없어 데이터를 다시 올리려면 운영 책임자에게 ZIP을 보내주세요.
- **공유 만료 알림** — `collaborator_shares.expires_at` 도래해도 별도 알림은 없습니다.

발견 즉시 알려주실수록 좋습니다. 어떤 채널이든 다음이 들어있으면 충분합니다.

1. **무엇을 하려고 했는지** (예: "야장 #2025_담양_03 사진 추가를 누르려고 했음")
2. **무엇을 보거나 들었는지** (에러 메시지, 화면 캡처, 또는 "버튼이 안 눌렸음")
3. **시각** (대략이라도 — 로그 추적용)
4. **단말·브라우저** (예: "iPhone 15, Safari" / "Android 14, Chrome")

## 5. 베타 참여자의 데이터

- 베타 중 입력하신 데이터는 **운영 환경**에 그대로 적재됩니다(임시 staging 이 아닙니다). 의미 있는 실데이터를 입력해주셔도 좋고, 시험용이라면 채취번호에 `TEST_` 같은 접두어를 붙여 구분해 주시면 정리가 쉽습니다.
- 운영 책임자(admin)는 audit_log 를 통해 누가 어떤 row 를 만들고 수정·삭제했는지 추적할 수 있습니다.

## 6. 시험 중 종료/탈퇴를 원하시면

운영 책임자에게 알려주시면 됩니다. `users_meta.active = false` 처리로 즉시 접근이 차단됩니다.
